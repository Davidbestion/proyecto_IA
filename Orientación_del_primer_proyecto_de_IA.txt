📜 Descripción del Juego HEX
HEX es un juego de estrategia para dos jugadores en un tablero hexagonal de tamaño ( N x
N ).

   Objetivo:
        Jugador 1 (🔴): Conectar los lados izquierdo y derecho (horizontal)
        Jugador 2 (🔵): Conectar los lados superior e inferior (vertical)
   Reglas:
    1. Los jugadores alternan turnos para colocar su ficha en una casilla vacía.
    2. El primer jugador en conectar sus lados gana.
    3. Conexiones válidas en las 6 direcciones del hexágono.
   Detalles de implementación:
    1. El tablero se modela como una matrix de NxN:
    2. El primer jugador tiene id = 1 y el segundo id = 2
    3. El sistema que se utiliza para modelación de adyacencias es even-r layout .




🛠️ Requerimientos del Proyecto
Desarrollar un jugador autónomo que herede de una clase base Player y decida la mejor
jugada usando IA.

   Heredar de la clase base Player .
   Usar los parámetros de play() para decidir la jugada:
         board : Matriz NxN que representa una COPIA del tablero actual ( 0 = vacío, 1 =
        tú, 2 = oponente).
         return : Tupla (fila, columna) con la jugada seleccionada.

📦 Formato de Entrega y Estructura
Para garantizar que su jugador pueda competir, deben seguir estrictamente esta estructura
de archivos:

   Directorio Raíz: Una carpeta nombrada Nombre_Apellido1_Apellido2.
   Archivo Principal: solution.py. Todo su código (clase, funciones auxiliares, constantes)
   debe estar en este archivo.
   Dependencias: requirements.txt. Solo si usan librerías externas (ej. numpy, scipy). Si no
   usan nada extra, no es necesario.
   Documentación: README.md o PDF. Breve explicación de la estrategia (ej: "MCTS con
   heurística de conexión").

🏆  Competición de Jugadores HEX: Reglas y Estructura del
Torneo
Los jugadores implementados se enfrentarán en un torneo automatizado con las siguientes
reglas:

 1. Emparejamientos Aleatorios
       Cada jugador competirá contra todos los demás en partidas múltiples.
       El orden y los oponentes se asignarán aleatoriamente al inicio.
 2. Partidas por Enfrentamiento:
       Cada duelo constará de N partidas (ej: 10) para reducir el factor suerte.
       Los jugadores alternarán quién empieza primero en cada partida.
       ¡Cuidado con los tiempos! Si tu algoritmo tarda mucho por jugada (+5 segundos),
       será descalificado.
